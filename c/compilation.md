# コンパイル(GCC)
## 工程
1. PreProcessing
2. Compilation
3. Assembly
4. Linking

## PreProcessing
### コマンド例
- `gcc -E main.c -o main.i`
  - `-E`: プリプロセスで止める
### 詳細
- `#include` で指定されたヘッダファイルの内容を、その場所に物理的に展開。
- `#define` マクロを定義に従って置換。
- `#if` などの条件付きコンパイルに基づき、不要なコード行を削除。
- コメントを削除。

## Compilation
### コマンド例
- `gcc -S main.i -o main.s`
  - `-S`: アセンブリ言語で止める
### 詳細
- 字句解析・構文解析: プリプロセス済みコードを解析し、抽象構文木 (AST) を構築。
- 最適化: 指定されたレベル（-O2等）に基づき、冗長な命令の削除やループの展開などを実行。
- コード生成: ターゲットとするCPUアーキテクチャ（x86_64, ARM等）に合わせたアセンブリ言語命令を生成。

## Assembly
### コマンド例
- `gcc -c main.s -o main.o`
  - `-c`: 機械語データで止める
### 詳細
- アセンブリ言語を、CPUが直接解読可能なバイナリ形式（機械語）に変換。
- この時点のファイルは ELF形式 (Linuxの場合) 等の構造を持ちますが、外部ライブラリ（printfなど）の関数アドレスはまだ確定しておらず、空欄（再配置情報）として残される。

## Linking
### コマンド例
- `gcc main.o -o main`
### 詳細
- シンボル解決: main.o 内で呼び出している printf などの実体が、どのライブラリのどの位置にあるかを特定し、アドレスを結びつける。
- 再配置: 各セクション（コード、データ等）をメモリ上の適切なアドレスに配置し直す。
- スタートアップコード（_start 等、main 関数を呼び出す前の前処理）を結合。
- 実行ファイルの拡張子は不要だが、デフォルトの`a.out`に合わせて、`.out`を付ける事が多い

# GCC
## 主要オプション
| **オプション** | **意味・役割** | **記述例・用途** |
| :--- | :--- | :--- |
| `-o <file>` | 出力ファイル（実行ファイル）の名前を指定する | `gcc main.c -o app` |
| `-c` | リンクを行わず、コンパイルのみ行いオブジェクトファイル（`.o`）を作成する | `gcc -c main.c` |
| `-Wall` | 一般的な警告（Warning）メッセージをすべて有効にする | `gcc -Wall main.c` |
| `-Wextra` | `-Wall` に含まれない、さらに詳細な警告メッセージを有効にする | `gcc -Wall -Wextra main.c` |
| `-g` | デバッグ情報（`gdb`などで使用）を実行ファイルに埋め込む | `gcc -g main.c` |
| `-O<n>` | コードの最適化レベルを指定する（`0`: なし, `1`〜`3`: 速度優先, `s`: サイズ優先） | `gcc -O2 main.c` |
| `-I<dir>` | ヘッダファイル（`.h`）を検索するディレクトリ（インクルードパス）を追加する | `gcc -I./include main.c` |
| `-l<lib>` | リンクするライブラリ（`lib<lib>.a` または `.so`）を指定する | `gcc main.c -lm` |
| `-L<dir>` | ライブラリファイルを検索するディレクトリを追加する | `gcc main.c -L./lib -lfoo` |
| `-std=<ver>` | 使用するC言語の標準規格を指定する（`c99`, `c11`, `c17` など） | `gcc -std=c11 main.c` |
| `-E` | プリプロセス（マクロ展開やヘッダ展開）のみ行い、結果を標準出力する | `gcc -E main.c` |
| `-v` | コンパイルの進捗状況や`gcc`のバージョン、内部で呼ばれるコマンドを表示する | `gcc -v main.c` |

## GCCバージョン別 デフォルトC言語規格 対応表
| GCCバージョン | デフォルトの規格オプション | 規格名 (通称) | 備考・主な変更点 |
| :--- | :--- | :--- | :--- |
| **4.x 以前** | `-std=gnu89` | **C89 / C90** | 1989年の規格。変数宣言をブロックの先頭で行う必要がある。 |
| **5.1 〜 7.x** | `-std=gnu11` | **C11** | 2011年の規格。マルチスレッド、`_Generic`、型ジェネリック数学などが導入。 |
| **8.1 〜 14.x** | `-std=gnu17` | **C17** | C11の不備を修正したマイナーアップデート（バグ修正版）。現在の主流。 |
| **15.x (開発中)** | `-std=gnu23` | **C23** | 最新規格。`bool`, `true`, `false`, `nullptr` がキーワードとして導入される。 |
